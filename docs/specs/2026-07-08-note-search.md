# Спека: поиск по заметкам — слайс 8

Дата: 2026-07-08 · Tracker: none (key-less)
Опирается на: просмотр/редактирование (VaultRepository, VaultViewModel, VaultListScreen).

## Цель
Дать искать заметки по запросу (по имени файла и содержимому) и открывать найденную.

## Границы (scope)
**В слайсе:** `VaultRepository.search`, `VaultViewModel.search` + состояние, поле поиска и
показ результатов в `VaultListScreen`.
**НЕ в слайсе:** индекс, ранжирование, подсветка/сниппеты совпадений, регэкспы, поиск по тегам.

## Проектные решения
- Совпадение = подстрока (регистр игнорируется) в имени файла ИЛИ в содержимом.
- Наивный полный скан содержимого без индекса — приемлемо для личного vault
  (`// ponytail: индекс, если станет медленно`).
- Пустой/пробельный запрос → результатов нет; UI при пустом запросе показывает весь список.

## Архитектура и интерфейсы

### VaultRepository (commonMain) — поиск
```kotlin
fun search(query: String): List<MdFile>
```
Пустой запрос → `emptyList()`; иначе `listMarkdownFiles().filter { name или readFile содержит
query, оба в lowercase }`.

### VaultViewModel (commonMain)
`VaultState` дополняется: `query: String = ""`, `results: List<MdFile> = emptyList()`.
```kotlin
fun search(query: String)   // на IO: _state.copy(query=…); results = repo.search(query)
```

### VaultListScreen (commonMain, `app.obsidianmd.ui`)
Добавляется поле поиска сверху (`OutlinedTextField`, `onSearch: (String) -> Unit`). Список
показывает `results`, когда `query` непустой, иначе все `files`. Тап открывает заметку
(существующий `onOpen`). Сигнатура дополняется параметрами `query`, `results`, `onSearch`.

## Приёмочные тест-кейсы (ручные, после разработки)
1. **Поиск по имени**
   - Изначальное состояние: vault с несколькими заметками, на экране списка.
   - Шаги: ввести часть имени файла в поле поиска.
   - Ожидаемый результат: список сузился до файлов, чьё имя содержит запрос.
2. **Поиск по содержимому**
   - Изначальное состояние: заметка содержит слово, которого нет в её имени.
   - Шаги: ввести это слово в поиск.
   - Ожидаемый результат: заметка присутствует в результатах; тап открывает её.
3. **Очистка запроса**
   - Изначальное состояние: введён запрос, показаны результаты.
   - Шаги: очистить поле поиска.
   - Ожидаемый результат: снова показан весь список заметок.
4. **Нет совпадений**
   - Изначальное состояние: на экране списка.
   - Шаги: ввести строку, которой нет ни в именах, ни в текстах.
   - Ожидаемый результат: пустой список результатов, приложение не падает.
