# Спека: создание md-файлов и папок в vault

> Fresh-run delivery-loop, `tracker: none`. Решения приняты по разумным умолчаниям
> (auto-approve), зафиксированы здесь. Приёмочные кейсы прогоняются вручную после разработки.

## Проблема
Vault можно только просматривать/редактировать/искать — создать новую заметку или папку
нельзя. Нужно создание прямо с экрана списка, в текущем открытом каталоге.

## Решение (обзор)
Кнопка «+» (FAB) на `VaultListScreen` → меню «Новая заметка» / «Новая папка» → диалог с полем
имени и валидацией. Создание идёт в **текущем каталоге** (`VaultState.currentDir`). Новая
заметка сразу открывается в редакторе; новая папка появляется в списке (папки — первыми).
Изменения на диске подхватывает обычная git-синхронизация (push при следующем sync).

### Разбивка по слоям (каждый юнит тестируется изолированно)

**1. Чистая логика имён — `:vault:api`, `app.obsidianmd.vault.EntryNaming` (новый файл)**
Без IO, чтобы тестировать в изоляции и переиспользовать в VM и в UI-диалоге.
- `fun noteFileName(raw: String): String` — `raw.trim()`, добавляет `.md`, если его нет
  (регистронезависимо). Для папок имя = `raw.trim()` (без расширения).
- `enum class NameError { Blank, Slash, Exists }`
- `fun entryNameError(finalName: String, existing: List<String>): NameError?` —
  `Blank` если пусто/только пробелы; `Slash` если содержит `/`; `Exists` если `finalName`
  уже есть в `existing` (сравнение без регистра); иначе `null`.
  Вызывающий сам приводит имя к финальному виду (заметка → `noteFileName`, папка → trim),
  затем проверяет коллизию против имён текущего каталога.

**2. Репозиторий — `:vault:api` + `OkioVaultRepository`**
- Новый метод интерфейса: `fun createFolder(path: String)`.
  Реализация: `fs.createDirectories(path.toPath())` (создаёт и родителей; идемпотентно).
- Создание заметки отдельного метода не требует — переиспользуем `writeFile(path, "")`
  (родитель = текущий каталог, он уже существует, т.к. мы его листаем).
- Оба фейка `FakeVaultRepository` (composeApp/commonTest, features/ai/impl/commonTest)
  получают `createFolder` (кладут маркерный ключ, чтобы папка появилась в `listEntries`).

**3. ViewModel — `VaultViewModel`**
- `fun createFolder(rawName: String)`: `path = "$currentDir/${rawName.trim()}"`;
  `launch { withContext(io) { repo.createFolder(path) }; loadDir(currentDir) }`.
- `fun createNote(rawName: String, onCreated: (String) -> Unit)`:
  `path = "$currentDir/" + noteFileName(rawName)`;
  `launch { withContext(io) { repo.writeFile(path, "") }; loadDir(currentDir); onCreated(path) }`.
  `onCreated` вызывается **после** записи (на главном диспетчере viewModelScope) — навигация к
  заметке идёт только когда файл уже на диске, без гонки read-before-write.
- `currentDir` берём из `_state.value.currentDir` (уже отслеживается).

**4. UI — `VaultListScreen` (+ `VaultPresentationProvider`)**
- FAB `Icons.Filled.Add` в слоте `floatingActionButton` у `Scaffold`. Скрыт в режиме поиска.
- Тап по FAB → `DropdownMenu` (два пункта: «Новая заметка», «Новая папка») — без новых
  зависимостей, якорь у FAB.
- Выбор пункта → `AlertDialog` с одним `OutlinedTextField` (имя) и кнопками «Создать»/«Отмена».
  Живая валидация через `entryNameError` по именам из `entries` текущего каталога:
  кнопка «Создать» неактивна при ошибке; под полем — текст ошибки (пусто / символ «/» / уже есть).
  Для заметки финальное имя = `noteFileName(text)`, для папки = `text.trim()`.
- На «Создать»: заметка → `onCreateNote(text)`; папка → `onCreateFolder(text)`; диалог закрыть.
- Новые коллбэки провайдера: `onCreateNote: (String) -> Unit`, `onCreateFolder: (String) -> Unit`.

**5. Навигация — `AppNavHost`**
- `onCreateNote = { name -> vm.createNote(name) { path -> backStack.add(Route.Note(path)) } }`.
- `onCreateFolder = vm::createFolder`.

### i18n
Новые строки в `values/strings.xml` и `values-ru/strings.xml` (секция «Notes list»):
`action_new_note`, `action_new_folder`, `action_create`, `cd_create` (FAB),
`create_note_title`, `create_folder_title`, `create_name_hint`,
`create_error_blank`, `create_error_slash`, `create_error_exists`.

### Что НЕ делаем (YAGNI)
- Автопереименование при коллизии («name 1») — вместо этого понятная ошибка, пользователь
  выбирает другое имя (меньше кода, поведение предсказуемо, ничего не затираем).
- Переименование/удаление/перемещение — вне задачи.
- Создание не-.md файлов, шаблоны заметок, вложенные пути в поле имени (символ «/» запрещён).

## Тестирование (TDD — сперва падающий тест)

- **`EntryNamingTest`** (`:vault:api` commonTest, чистая логика):
  - первый падающий тест: `noteFileName("todo")` == `"todo.md"`; затем `"a.md"`→`"a.md"`,
    trim пробелов, `"A.MD"` не удваивает расширение.
  - `entryNameError`: `""`/`"  "`→`Blank`; `"a/b"`→`Slash`; `"note.md"` при `["note.md"]`→`Exists`
    (без регистра); валидное новое имя→`null`.
  - минимальный код: две функции + enum.
- **`VaultRepositoryTest`** (okio `FakeFileSystem`):
  - первый падающий тест: после `createFolder("$root/Ideas")` в `listEntries(root)` есть
    запись `Ideas` c `isFolder=true`; повторный вызов не падает (идемпотентность);
    вложенный путь создаёт родителей.
- **`VaultViewModelTest`** (`FakeVaultRepository`):
  - `createFolder("Ideas")` → в репозитории появилась папка, `state.entries` содержит её после refresh.
  - `createNote("todo")` → записан пустой файл `"$root/todo.md"`, `onCreated` получил этот путь,
    список обновлён.
- **`VaultListScreenTest`** (Robolectric `compose-uiTest`):
  - FAB виден вне поиска; тап → видно пункты меню «Новая заметка»/«Новая папка».
  - в диалоге создания при пустом имени и при дубликате кнопка «Создать» неактивна.

Каждый юнит изолирован: логика имён — чистая (без FS), репозиторий — на `FakeFileSystem`,
VM — на `FakeVaultRepository`, экран — на фейковых коллбэках. Интерфейсы не требуют поднимать
остальную систему.

## Приёмочные тест-кейсы (ручной прогон после разработки)

**Создание новой заметки открывает редактор**
- Изначальное состояние: приложение открыто на списке заметок (корень vault).
- Шаги: нажать «+» → «Новая заметка» → ввести `Идеи` → «Создать».
- Ожидаемый результат: создан файл `Идеи.md`, открылся пустой редактор заметки; после выхода
  назад `Идеи.md` виден в списке.

**Создание папки и вход в неё**
- Изначальное состояние: список заметок (корень vault).
- Шаги: нажать «+» → «Новая папка» → ввести `Проекты` → «Создать» → тапнуть по папке `Проекты`.
- Ожидаемый результат: папка `Проекты` появилась в списке первой (папки выше файлов), открылась
  пустой каталог `Проекты`.

**Создание внутри подпапки**
- Изначальное состояние: пользователь зашёл в папку `Проекты`.
- Шаги: нажать «+» → «Новая заметка» → ввести `план` → «Создать».
- Ожидаемый результат: заметка создана внутри `Проекты` (`Проекты/план.md`), открылась в редакторе.

**Валидация имени**
- Изначальное состояние: в корне уже есть заметка `Идеи.md`; открыт диалог «Новая заметка».
- Шаги: (а) оставить поле пустым; (б) ввести `Идеи`; (в) ввести `a/b`.
- Ожидаемый результат: во всех трёх случаях кнопка «Создать» неактивна, под полем — понятная
  ошибка (пусто / уже существует / недопустимый символ «/»); валидное новое имя ошибку убирает.

**Синхронизация подхватывает созданное**
- Изначальное состояние: настроен git-репозиторий с доступом на запись; создана новая заметка.
- Шаги: выполнить синхронизацию (Settings → Sync now или pull-to-refresh).
- Ожидаемый результат: новый файл уходит в удалённый репозиторий (виден в истории git на сервере).
