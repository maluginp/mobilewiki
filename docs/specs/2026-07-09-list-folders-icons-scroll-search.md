# Спека: папки + красота списка + скрытие поиска по скроллу

## Решения
- **Папки.** `VaultRepository.listEntries(dir)` возвращает `VaultEntry(name, path, isFolder)`:
  подпапки + `.md`-файлы этого каталога, сортировка — папки первыми, затем по имени
  (без регистра). Навигация в VaultViewModel: `openFolder`, `upFolder`, флаг `atRoot`.
  Поиск остаётся плоским по корню (вне scope). `listMarkdownFiles()` сохраняется для
  синка/поиска и старых тестов.
- **Красота.** Строки — Material3 `ListItem` с ведущей иконкой (`Folder` для папок,
  `Description` для файлов). У файлов в заголовке скрыт суффикс `.md`.
- **Скролл-поиск.** `nestedScroll`: аккумулируем дельту скролла в offset панели поиска
  (клампим в `[-height, 0]`), панель сдвигаем на offset, верхний `contentPadding`
  списка = `height + offset` — движение 1:1 со скроллом, вверх — панель сразу
  возвращается. Плавно, без анимационных «прыжков».

## Приёмочные тест-кейсы (ручной прогон)
- **Папки видны и открываются.** Изн. состояние: vault с подпапкой (напр. `Daily/`)
  и файлами в корне. Шаги: открыть список → тапнуть папку → тапнуть «Назад».
  Ожидаемо: папка видна в списке (иконка папки), тап открывает её содержимое,
  «Назад» возвращает в корень.
- **Иконки и заголовки.** Изн.: есть папка и `note.md`. Шаги: посмотреть список.
  Ожидаемо: у папки — иконка папки и её имя; у файла — иконка документа и «note»
  (без `.md`).
- **Поиск прячется/появляется по скроллу.** Изн.: заметок больше экрана. Шаги:
  проскроллить вниз, затем вверх. Ожидаемо: при скролле вниз поле поиска плавно
  уезжает вверх синхронно с контентом; при скролле вверх — плавно возвращается.
