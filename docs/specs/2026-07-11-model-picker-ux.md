# Спека — UX выбора AI-модели

## Решение

Заменить inline-автокомплит на **отдельный экран выбора модели**, встроенный в существующую
навигацию App.kt (ещё один булев флаг `showModelPicker`, как `showSettings`/`showAi`).

### Настройки (SettingsScreen)
- В секции AI вместо `ModelField` — строка: **имя текущей модели** (или «Not selected») +
  иконка `Edit` (`IconButton`), по клику → `onEditModel()`.
- Модель больше не сохраняется по кнопке «Save» — она персистится сразу при выборе на экране
  списка (`setAiModel`). «Save» сохраняет только URL и ключ.

### Экран списка моделей (ModelPickerScreen)
- В AppBar (централизован в App.kt): заголовок «AI model», стрелка «Назад» (закрывает экран),
  иконка `Search` → переключает поле поиска прямо в AppBar (паттерн как у поиска по заметкам).
- Загрузка (список пуст и `loading`): по центру `CircularProgressIndicator` (вертелка).
- **Pull-to-refresh**: `PullToRefreshBox` (Material3 1.7), `isRefreshing = loading`,
  `onRefresh` → форс-перезагрузка списка (`reloadModels()`).
- Строка модели (`ListItem`): headline = `name`, supporting = `id` + размер контекста +
  цена промпта; выбранная модель помечена галочкой (`Check`). Тап → `onSelect(id)` +
  закрытие экрана.
- Данные в строке: **имя**, **id (slug)**, **контекст** («128K ctx»), **цена входа**
  («$0.15/M») — из полей OpenRouter `name/id/context_length/pricing.prompt`.

### Модель данных
`ModelInfo` расширяется: `contextLength: Long?`, `pricing: ModelPricing?`
(`prompt`, `completion` — строки-цены за токен, как отдаёт API). Чистые форматтеры
(`ModelInfo.contextLabel()`, `ModelInfo.priceLabel()`) — покрыты юнит-тестами.

### ViewModel
`SettingsViewModel.reloadModels()` — форс-перезагрузка (сбрасывает кэш-гвард, грузит заново).
`loadModels()` остаётся для ленивой первой загрузки при включении AI / открытии пикера.

## Не делаем (YAGNI)
- Никакой навигационной библиотеки — остаёмся на булевых флагах App.kt.
- Не показываем полное описание модели / модальности — только 4 ключевых поля в строке.

## Приёмочные тест-кейсы (ручные)

1. **Просмотр и открытие**
   - Изначально: AI включён, модель выбрана (напр. `openai/gpt-4o-mini`).
   - Шаги: открыть Settings → секция AI.
   - Ожидание: видно имя модели и иконку карандаша; поля-автокомплита нет.
   - Шаги: тап по карандашу → открывается экран «AI model» со списком.

2. **Загрузка (вертелка)**
   - Изначально: список ещё не загружен (сеть медленная / первый заход).
   - Шаги: открыть экран моделей.
   - Ожидание: по центру крутится индикатор, затем появляется список.

3. **Поиск**
   - Шаги: на экране моделей тап по иконке «Поиск» → появляется поле → ввести «claude».
   - Ожидание: список отфильтрован по имени/slug; крестик закрывает поиск.

4. **Pull-to-refresh**
   - Шаги: потянуть список вниз.
   - Ожидание: индикатор обновления, список перезагружается с OpenRouter.

5. **Выбор**
   - Шаги: тап по строке модели.
   - Ожидание: экран закрывается, в настройках показана выбранная модель; после перезапуска
     AI-чата используется именно она.

6. **Данные в строке**
   - Ожидание: у моделей видно имя, slug, размер контекста и цену входа (у кого есть).
