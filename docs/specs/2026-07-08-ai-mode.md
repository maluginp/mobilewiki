# Спека: AI-режим на OpenRouter — слайс 9

Дата: 2026-07-08 · Tracker: none (key-less)
Опирается на: Ktor/serialization (OAuth-слайс), VaultRepository (list/read/write/search),
паттерн секретов (EncryptedSharedPreferences), суспенд-мост UiConflictResolver.

## Цель
AI помогает работать с заметками через OpenRouter: отвечает на вопросы, ищет и читает заметки,
предлагает создание/правку заметок (применяются только после подтверждения пользователя).

## Границы (scope) — полный AI-режим (одобрено)
**В слайсе:** `ApiKeyStore` (ключ OpenRouter), `OpenRouterClient` (Ktor, tool-calling),
инструменты `search_notes`/`read_note`/`write_note`, `AiAgent` (цикл с подтверждением записи),
`AiViewModel`, `AiChatScreen` + диалог подтверждения, поле ключа в настройках, вход «AI».
**НЕ в слайсе:** стриминг ответов, история между запусками, выбор модели в UI, вложения/картинки,
удаление/переименование заметок AI.

## Проектные решения
- **Безопасность:** `write_note` НИКОГДА не применяется автоматически — только через
  `WriteApprover.confirm(...)` (suspend-мост к диалогу, как `UiConflictResolver`). read-only
  инструменты (`search_notes`, `read_note`) исполняются автоматически.
- **Модель:** константа по умолчанию `openai/gpt-4o-mini` (в UI не выносим — YAGNI).
- **Ключ:** секрет → EncryptedSharedPreferences (файл `obsidian_secure`, ключ `openrouter_key`).
- **Лимит цикла:** не более 5 итераций tool-loop (защита от зацикливания).

## Архитектура и интерфейсы (пакет `app.obsidianmd.ai`, commonMain)

### DTO (`@Serializable`)
Модель OpenRouter chat completions:
```kotlin
data class ChatMessage(role: String, content: String? = null,
                       toolCalls: List<ToolCall>? = null, toolCallId: String? = null, name: String? = null)
data class ToolCall(id: String, function: FunctionCall)   // type="function"
data class FunctionCall(name: String, arguments: String)  // arguments — JSON-строка
data class ChatRequest(model, messages, tools)            // tools — описания функций
data class ChatResponse(choices: List<Choice>)            // choices[0].message
```
(@SerialName для snake_case: `tool_calls`, `tool_call_id`, …)

### ApiKeyStore
```kotlin
interface ApiKeyStore { fun getKey(): String?; fun saveKey(key: String) }
```
Android: `EncryptedApiKeyStore` (EncryptedSharedPreferences). Тест: `FakeApiKeyStore`.

### OpenRouterClient
```kotlin
class OpenRouterClient(http: HttpClient, apiKey: String) {
    suspend fun chat(messages: List<ChatMessage>): ChatResponse   // model=константа, tools=встроенные описания
}
```
POST `https://openrouter.ai/api/v1/chat/completions`, заголовок `Authorization: Bearer <apiKey>`.
Тестируется на MockEngine.

### Инструменты и AiAgent
```kotlin
fun interface WriteApprover { suspend fun confirm(name: String, content: String): Boolean }

class AiAgent(client: OpenRouterClient, repo: VaultRepository, approver: WriteApprover, maxSteps: Int = 5) {
    suspend fun ask(history: List<ChatMessage>): AiResult   // AiResult.Answer(text) | AiResult.Failed(reason)
}
```
Цикл: `client.chat(messages)` → если `tool_calls`:
- `search_notes(query)` → `repo.search` → имена; `read_note(name)` → `repo.readFile`; результат
  добавляется как tool-сообщение, цикл продолжается;
- `write_note(name, content)` → `approver.confirm(...)`; при `true` — `repo.writeFile` и
  tool-результат «сохранено», при `false` — «отклонено пользователем»; цикл продолжается;
иначе (есть `content`) → `AiResult.Answer(content)`. Превышен `maxSteps` → `Failed`.

### AiViewModel
Держит `messages`, `status` (Idle/Thinking/Answered/Failed), `pendingWrite` (для диалога).
`send(text)` запускает `AiAgent.ask`; `approveWrite`/`rejectWrite` разблокируют approver
(тот же приём, что `resolveConflict`).

### UI
- `AiChatScreen(messages, input, onSend, ...)` — лента реплик + поле ввода + «Отправить».
- Диалог подтверждения записи при `pendingWrite != null` (имя + превью контента,
  «Применить»/«Отклонить»).
- Вход: кнопка «AI» на экране списка (`onOpenAi`), навигация в `App` (как `showSettings`).
- Настройки: поле «Ключ OpenRouter» в `SettingsScreen`, сохраняется в `ApiKeyStore`.

### Android-проводка
`EncryptedApiKeyStore`, `OpenRouterClient` на общем `HttpClient` (OkHttp), сборка `AiViewModel`
в `MainActivity`; кнопка входа «AI».

3. **AiAgent (фейк-клиент со скриптованными ответами + FakeVaultRepository/фейк-approver):**
   - модель зовёт `search_notes` → агент исполняет через repo, докладывает, на след. шаге модель
     даёт финальный ответ → `Answer`;
   - `write_note` + approver `true` → `repo.writeFile` вызван, ответ содержит успех;
   - `write_note` + approver `false` → `writeFile` НЕ вызван;
   - превышение `maxSteps` → `Failed`.
4. **AiViewModel:** `send` → статус `Thinking`→`Answered`; при write модель → `pendingWrite`
   выставлен, `approveWrite()` разблокирует и завершает.

## Приёмочные тест-кейсы (ручные, после разработки)
1. **Вопрос-ответ**
   - Изначальное: задан ключ OpenRouter; открыт экран AI.
   - Шаги: спросить «привет, что ты умеешь?».
   - Ожидаемый результат: приходит текстовый ответ модели.
2. **Поиск через AI**
   - Изначальное: в vault есть заметка со словом X; экран AI.
   - Шаги: «найди мои заметки про X».
   - Ожидаемый результат: AI вызывает поиск и отвечает, упомянув найденную заметку.
3. **Создание заметки с подтверждением**
   - Изначальное: экран AI; ключ задан.
   - Шаги: «создай заметку today.md с текстом …» → в диалоге нажать «Применить».
   - Ожидаемый результат: файл создан в vault (виден в списке); при синке уходит на сервер.
4. **Отклонение записи**
   - Изначальное: AI предложил запись, показан диалог.
   - Шаги: «Отклонить».
   - Ожидаемый результат: файл не создан/не изменён; AI сообщает, что действие отклонено.
5. **Нет ключа**
   - Изначальное: ключ OpenRouter не задан.
   - Шаги: отправить сообщение в AI.
   - Ожидаемый результат: понятная ошибка «ключ не задан», приложение не падает.
