# Спека: инструкция по авторизации в GitHub

## Проблема
В `LoginScreen`, состояние `AuthState.AwaitingUser`, показаны код и кнопки, но нет
объяснения последовательности действий. Пользователь не понимает, что делать.

## Решение
Добавить строку `login_instructions` с пошаговой инструкцией и показать её в состоянии
`AwaitingUser` (над спиннером/«Waiting…»). Формулировка: скопировать код → «Open GitHub» →
вставить код на GitHub → подтвердить; экран продолжит сам. В состоянии `Idle` инструкции нет.

Правки только UI-строки + вёрстка `AwaitingUser`. Логику device flow не трогаем.

## Приёмочные тест-кейсы

**Инструкция видна при ожидании подтверждения**
- Изначальное состояние: запущена авторизация, GitHub вернул user code (AwaitingUser).
- Шаги: открыть экран входа → нажать «Sign in with GitHub» → дождаться кода.
- Ожидаемый результат: под кодом видна пошаговая инструкция (скопировать код, открыть
  GitHub, вставить код, подтвердить).

**Инструкции нет до старта**
- Изначальное состояние: экран входа, авторизация ещё не запущена (Idle).
- Шаги: открыть экран входа.
- Ожидаемый результат: видна только кнопка «Sign in with GitHub», инструкции нет.
