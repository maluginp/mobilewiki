# Spec — UX-friendly UI для AI-чата

## Решение

Переписать `AiChatScreen` на Material3-паттерн чата и заменить иконку AI в нижней навигации.
Общий визуальный знак AI — `Icons.Filled.AutoAwesome` (иконка «AI»), используется и в аватаре,
и в нижней навигации. Контракт компонента и вызов из `App.kt` не меняются.

### Элементы
1. **Пузыри сообщений** (`MessageBubble`):
   - user — выравнивание справа, `colorScheme.primary` фон / `onPrimary` текст, без аватара;
   - assistant — слева, круглый аватар (`AutoAwesome` на `primaryContainer`) + пузырь
     `surfaceVariant` / `onSurfaceVariant`;
   - скруглённые углы (18.dp, «хвостик» — уменьшенный угол со стороны отправителя), max ширина ~82%.
2. **Индикатор набора** (`TypingIndicator`): аватар + пузырь с тремя анимированными точками,
   показывается при `AiStatus.Thinking`. Помечается `testTag("typing")`.
3. **Ошибка**: карточка `errorContainer` с текстом причины (существующий `error_with_reason`).
4. **Поле ввода** (`ChatInputBar`): скруглённое поле + круглая кнопка отправки с
   `Icons.AutoMirrored.Filled.Send`; кнопка неактивна при пустом вводе или `Thinking`.
5. **Пустой экран** (`EmptyState`): крупный аватар + приветствие, когда сообщений нет.
6. **Автопрокрутка**: `LazyColumn` листается к последнему элементу при добавлении сообщения /
   переходе в Thinking.
7. **Нижняя навигация** (`App.kt`): иконка AI-пункта — `AutoAwesome` вместо `Face`.

### Новые строковые ресурсы
`chat_empty_title`, `chat_empty_subtitle`, `chat_input_hint`, `cd_ai_avatar`, `cd_send`.
Оставить `chat_you_prefix`/`chat_ai_prefix`? Нет — префиксы заменены аватаром/выравниванием, удалить.

### Вне объёма
Стриминг ответа по токенам, история/сессии, разметка markdown в пузырях, launcher-иконка приложения.

## Приёмочные тест-кейсы

### 1. Пузыри и аватар
- Изначальное состояние: AI включён, открыт экран AI-чата, есть пара реплик (user + assistant).
- Шаги: посмотреть на переписку.
- Ожидаемый результат: реплика пользователя — справа, реплика AI — слева и с круглым аватаром;
  у пузырей разный фон; префиксов «You:/AI:» нет.

### 2. Индикатор набора
- Изначальное состояние: экран чата открыт.
- Шаги: отправить сообщение и смотреть, пока идёт ответ.
- Ожидаемый результат: вместо строки «AI is thinking…» появляется пузырь с анимированными
  точками рядом с аватаром; после ответа он исчезает и появляется пузырь ответа.

### 3. Кнопка отправки
- Изначальное состояние: поле ввода пустое.
- Шаги: посмотреть на кнопку; ввести текст; отправить.
- Ожидаемый результат: при пустом поле и во время ответа кнопка отправки неактивна; при
  наличии текста — активна; после отправки поле очищается, сообщение уходит.

### 4. Пустой экран
- Изначальное состояние: AI включён, сообщений ещё нет.
- Шаги: открыть AI-чат.
- Ожидаемый результат: по центру — аватар и приветственный текст, а не пустое поле.

### 5. Иконка в нижней навигации
- Изначальное состояние: AI включён.
- Шаги: посмотреть на нижнюю навигацию.
- Ожидаемый результат: пункт AI отмечен иконкой-«AI» (AutoAwesome), не безликим лицом; вход
  в чат работает как раньше.

### 6. Подтверждение записи
- Изначальное состояние: AI предлагает записать файл (`pendingWrite`).
- Шаги: увидеть диалог; нажать Apply / Reject.
- Ожидаемый результат: диалог показывает имя файла и превью; Apply/Reject работают как раньше.
