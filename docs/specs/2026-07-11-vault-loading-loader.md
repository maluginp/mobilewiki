# Спека: лоадер при загрузке списка репозитория

## Проблема
`VaultViewModel.loadDir()` не выставляет `loading`, поэтому во время первичной загрузки
(`refresh()` на старте) `VaultListScreen` рисует пустое состояние «No files» вместо лоадера.

## Решение
1. `loadDir()` выставляет `loading = true` перед чтением с IO и `loading = false` после.
2. `VaultListScreen`: если `state.loading` и список пуст — `CircularProgressIndicator`;
   «No files» — только когда `!loading` и список пуст (как в `ModelPickerScreen`).

Поиск не трогаем: `loading` относится к загрузке папки, при поиске список догоняет отдельно.

## Приёмочные тест-кейсы

**Лоадер вместо «No files» при загрузке**
- Изначальное состояние: репозиторий уже выбран, приложение открывается заново.
- Шаги: открыть приложение → дождаться загрузки списка.
- Ожидаемый результат: пока список грузится, виден индикатор загрузки, а не «No files»;
  после загрузки — список файлов.

**«No files» для реально пустого репозитория**
- Изначальное состояние: выбран пустой репозиторий (без .md-файлов).
- Шаги: открыть приложение → дождаться окончания загрузки.
- Ожидаемый результат: после завершения загрузки показано «No files».
