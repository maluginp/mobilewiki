# Спека: UX редактирования заметок

Дата: 2026-07-10 · Tracker: none (key-less) · Ветка: text-editing-ux
Опирается на: редактирование docs/specs/2026-07-08-markdown-editing.md.

## Цель
Сделать правку заметок удобнее и безопаснее: панель форматирования над клавиатурой,
защита от потери несохранённых изменений, чистый полноэкранный редактор без рамки.

## Границы (scope)
**В слайсе:** чистые функции форматирования markdown над (текст+выделение); панель
инструментов в режиме правки; переход редактора со `String` на `TextFieldValue` (нужно
выделение); диалог подтверждения выхода с несохранёнными правками + индикатор «есть
несохранённое»; замена OutlinedTextField на полноэкранный BasicTextField.
**НЕ в слайсе:** живое превью при редактировании; создание новых заметок; undo/redo;
таблицы/картинки-вставка.

## Архитектура и интерфейсы

### 1. Чистая логика форматирования (commonMain, `app.obsidianmd.editor`)
```kotlin
// Мини-модель поля ввода без зависимости от Compose (тестируется в commonTest).
data class EditState(val text: String, val selStart: Int, val selEnd: Int)

object MdEdit {
    // Обернуть выделение маркером (bold="**", italic="*"); нет выделения → вставить
    // marker+marker и поставить каретку между ними.
    fun wrapInline(s: EditState, marker: String): EditState

    // Добавить префикс в начало строки, где стоит каретка ("# ", "- ", "- [ ] ").
    fun linePrefix(s: EditState, prefix: String): EditState

    // Ссылка: есть выделение → "[выд]()" каретка между "()"; нет → "[]()" каретка между "[]".
    fun link(s: EditState): EditState
}
```
Правила выделения после операции:
- `wrapInline` с выделением: новое выделение охватывает исходный текст внутри маркеров
  (`selStart+marker.length … selEnd+marker.length`).
- `wrapInline` без выделения: каретка (collapsed) между вставленными маркерами.
- `linePrefix`: каретка/выделение сдвигаются на `prefix.length` (префикс вставлен в начало
  текущей строки — по последнему `\n` до `selStart`).
- `link`: каретка ставится в место ввода URL (с выделением) или ярлыка (без выделения).

### 2. Панель форматирования (commonMain, `app.obsidianmd.ui`)
`EditorToolbar(onAction: (MdEdit.() -> EditState трансформация))` — Row из IconButton:
FormatBold, FormatItalic, Title, FormatListBulleted, CheckBox, Link (material-icons-extended).
Каждая кнопка отдаёт наверх нужную трансформацию над текущим `EditState`.

### 3. Редактор (правка в `MarkdownScreen`)
- Режим правки держит локальный `TextFieldValue` (текст + `selection`), инициализируется из
  входного `draft` один раз при входе в правку; на изменение — `onDraftChange(text)` наверх
  (App продолжает хранить `draft: String` для сохранения).
- Ввод — `BasicTextField` на весь экран (без рамки), `Modifier.fillMaxWidth().weight(1f)`,
  padding 16dp, читабельный `TextStyle` (bodyLarge), курсор виден.
- Снизу (над клавиатурой) — `EditorToolbar`; тап конвертирует `TextFieldValue`→`EditState`,
  применяет трансформацию, конвертирует обратно (с новым `TextRange`) и обновляет поле.

### 4. Безопасное сохранение (App.kt)
- `dirty = editing && draft != state.content`.
- Иконка ✓ (сохранить) в AppBar активна только при `dirty` (иначе приглушена/выключена).
- «Назад» в режиме правки: при `dirty` → диалог `UnsavedDialog` (Сохранить / Не сохранять /
  Отмена); без `dirty` → просто выйти из правки. Сохранить → `vm.saveFile` + выйти; Не
  сохранять → выйти без записи; Отмена → закрыть диалог.

## Тестирование (TDD)

### commonTest (чистая логика `MdEdit`)
1. **wrapInline с выделением.** *Первый падающий тест:* `EditState("abc",0,3)`, marker="**"
   → text="**abc**", выделение (2,5). Падает — класса нет.
2. **wrapInline без выделения.** `("",0,0)`, "**" → text="****", каретка (2,2).
3. **wrapInline курсор в середине.** `("ab",1,1)`, "*" → "a**b"? нет: вставка "**" на позиции 1
   → "a**b" неверно; ожидаем "a"+"*"+"*"+"b" = "a**b", каретка (2,2).
4. **linePrefix первая строка.** `("hello",0,0)`, "# " → "# hello", каретка (2,2).
5. **linePrefix на второй строке.** `("a\nbc",2,2)`, "- " → "a\n- bc", каретка (4,4).
6. **linePrefix checkbox.** префикс "- [ ] " добавляется в начало строки.
7. **link с выделением.** `("term",0,4)` → "[term]()", каретка между "()" (позиция 7,7).
8. **link без выделения.** `("",0,0)` → "[]()", каретка между "[]" (позиция 1,1).

Минимальный код на каждый — реализация соответствующей функции.

## Приёмочные тест-кейсы (ручные, после разработки)

1. **Форматирование выделения (bold)**
   - Изначальное состояние: заметка открыта, режим правки, есть текст.
   - Шаги: выделить слово → нажать Bold на панели.
   - Ожидаемый результат: слово обёрнуто в `**…**`, выделение осталось на слове (внутри
     маркеров); повторный просмотр показывает жирный.

2. **Вставка маркеров без выделения**
   - Изначальное состояние: режим правки, каретка в пустом месте, ничего не выделено.
   - Шаги: нажать Italic.
   - Ожидаемый результат: вставлены маркеры italic `*` + `*` (для bold было бы `**`+`**`),
     каретка между ними, можно сразу печатать курсивный текст.

3. **Заголовок / список / чекбокс (префикс строки)**
   - Изначальное состояние: режим правки, каретка в строке с текстом.
   - Шаги: нажать Heading, затем List, затем Checkbox (на разных строках).
   - Ожидаемый результат: в начало строки добавляются `# `, `- `, `- [ ] ` соответственно.

4. **Ссылка**
   - Изначальное состояние: режим правки, выделено слово.
   - Шаги: нажать Link.
   - Ожидаемый результат: `[слово]()`, каретка внутри `()`, можно вписать URL.

5. **Защита от потери правок при выходе**
   - Изначальное состояние: режим правки, внесены изменения (не сохранены).
   - Шаги: нажать «Назад».
   - Ожидаемый результат: показан диалог с выбором Сохранить / Не сохранять / Отмена;
     «Отмена» возвращает в редактор без потерь; «Не сохранять» выходит без записи;
     «Сохранить» пишет файл и выходит.

6. **Выход без изменений**
   - Изначальное состояние: режим правки, ничего не менялось.
   - Шаги: нажать «Назад».
   - Ожидаемый результат: сразу выход в просмотр, без диалога.

7. **Индикатор несохранённого**
   - Изначальное состояние: режим правки.
   - Шаги: посмотреть на иконку сохранения до и после ввода символа.
   - Ожидаемый результат: до изменений ✓ неактивна/приглушена, после — активна.

8. **Чистый редактор**
   - Изначальное состояние: вход в режим правки.
   - Шаги: осмотреть поле ввода.
   - Ожидаемый результат: нет рамки-прямоугольника, поле на весь экран, читабельный шрифт,
     нормальные отступы; ввод и прокрутка длинного текста работают.
