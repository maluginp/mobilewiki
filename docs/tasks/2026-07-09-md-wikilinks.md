<intent>
В режиме просмотра markdown ссылки вида [[...]] должны быть кликабельными и вести
к нужному файлу vault (только если файл найден), а картинки-эмбеды отображаться.
</intent>

<context>
KMP/Compose (только androidTarget) просмотрщик Obsidian-vault. Просмотр md —
MarkdownScreen через com.mikepenz multiplatform-markdown-renderer (0.27.0).
Обычный markdown-синтаксис [[wikilink]] библиотека не понимает — рендерит как текст.
Навигация к файлу — VaultViewModel.open(MdFile); список/папки — VaultRepository.
Внешних загрузчиков картинок (coil) в проекте нет.
</context>

<requirements>
- `[[file]]` — найти `file.md` в корне и во всех папках; клик открывает файл.
- `[[path/file]]` — найти по пути path и файлу `file.md`; клик открывает файл.
- `[[file.md]]` — найти файл с конкретным расширением (тоже в корне/папках).
- `![[file.png]]` — если это картинка (по расширению), отобразить её прямо в просмотрщике.
- Кликабельно/встраивается только если файл реально найден; иначе — остаётся обычным текстом.
</requirements>
