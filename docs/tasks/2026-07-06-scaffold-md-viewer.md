# Скаффолд KMP/Compose + просмотр md-файлов

Tracker: none (key-less)

<intent>
Заложить основу Android-приложения для Obsidian и дать пользователю просматривать
markdown-файлы из локального хранилища (vault): список файлов и отрендеренное
содержимое выбранного .md.
</intent>

<requirements>
- Проект собирается: Kotlin Multiplatform + Jetpack Compose (Android-таргет), корутины.
- Экран со списком .md-файлов в выбранной/захардкоженной для начала папке-vault.
- Тап по файлу открывает экран просмотра с отрендеренным markdown (заголовки, списки,
  жирный/курсив, ссылки, код — базовый набор).
- Чтение файлов не блокирует UI (корутины / Dispatchers.IO).
- Есть покрытие тестами на не-UI логику (парсинг/листинг файлов) — TDD.
</requirements>

<context>
Это ПЕРВЫЙ слайс большого продукта — Android-приложение для Obsidian с синхронизацией в
GitHub. Полный продукт (не в этом цикле): git-синхронизация, редактирование, поиск,
AI-режим на OpenRouter. Здесь — только фундамент проекта + просмотр md.

Стек (задан пользователем): Kotlin Multiplatform, Jetpack Compose, Coroutines.

Репозиторий на старте пустой (только .git + docs/). Нужен полный скаффолд Gradle-проекта.

Flow: локально, без MR (git remote не настроен). Цикл завершается на зелёном коде в
фиче-ветке.
