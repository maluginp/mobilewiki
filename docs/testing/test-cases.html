Brainer — тест-кейсы для ручного тестирования

Приложение app.obsidianmd (KMP/Compose). Автотесты — Maestro, прогнаны на Pixel 9a. Как запускать: maestro/README.md.

Всего кейсов: 38 Автоматизировано: 14 Доступно к автоматизации: 8 Только вручную: 16
✅ Да — есть Maestro-флоу (проверено). ⚙️ Доступно — можно автоматизировать, пока вручную. ❌ Нет — только вручную (внешняя авторизация/сеть/git/AI).

  2. Запустить приложение.
Экран «Welcome to Brainer» с тремя вариантами: Sign in with GitHub, Connect any git repository, Use locally (no sync).⚙️ Доступно

  1. «Sign in with GitHub».

  3. На GitHub вставить код, подтвердить.

  1. Войти через GitHub.

  2. Ввести URL и (опц.) токен.

TC-OB-07Инструкция авторизации на экране кода
  1. Начать GitHub-вход, дойти до экрана кода.
Пошаговая инструкция: скопировать код → открыть GitHub → вставить → подтвердить.❌ Нетчасть device flow

TC-RC-04«Назад» с ввода URL возвращает в настройки
  1. Settings → «Change repository» → «Enter manually» → стрелка «назад».
Экран «Enter a repository URL» показывает стрелку «назад»; нажатие возвращает в «Settings». «Назад» с пикера GitHub (с токеном) — вручную.✅ Даchange-repo-back.yaml
TC-RC-05«GitHub» без авторизации ведёт на вход
  1. Локальный режим.
  2. Settings → «Change repository» → «GitHub».
Показывается экран входа «Sign in with GitHub», а не пустой/ошибочный список репозиториев; после входа откроется выбор репозитория.✅ Даchange-repo-github-auth.yaml
TC-RC-03Переключение на локальный режим
  1. На экране смены выбрать «Local (no sync)».
Стор в локальный режим, возврат в настройки; текущий репозиторий — «Local storage (no sync)».⚙️ Доступно
TC-AI-01Включение AI раскрывает поля
  1. Settings.
  2. Тумблер «Enable AI».
Появляется поле API-ключа и выбор модели; в навигации появляется AI.⚙️ Доступнотап по Switch хрупок
TC-AI-02Выбор AI-модели
  1. Включить AI.
  2. Открыть пикер, применить фильтры/сортировку, pull-to-refresh.
Список моделей фильтруется/сортируется; модель сохраняется при выборе.❌ Нетсеть и модели OpenRouter
